# Version Control

## Git hosted on Github

- Main branch is stable, most up to date version
- All new developments are done within feature branches (branches are named by purpose)

- Pull request (PR) workflow is utilized, where each PR is reviewed and approved
- Before creating a PR, the latest main is pulled to decrease conflicts

- Merge commits are used to prevent overlap allowing for a clean commit log
- Conflicts are resolved locally before merging

- All while logging a history through commits
