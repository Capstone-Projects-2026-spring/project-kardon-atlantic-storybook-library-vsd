# Use-case descriptions

Use Case 1 - Account Creation (User / Caretaker)

*As a user, I want to create an account so I can save and manage my storybooks.*
  1. The user opens the application
  2. The user selects “Register” on the landing page.
  3. The user enters username, email, and password
  4. The system checks the input and creates the account.
  5. The system confirms registration.
  6. The user is redirected to their home/library page/.

Use Case 2 - Signing In (User / Caretaker)

*As a user, I want to log into my account so I can access my saved storybooks.*
  1. The user opens the application
  2. The user selects the 'Login' button to access the sign-in page.
  3. The user enters their email address and password.
  4. If the credentials are valid, The user is redirected to the home page where they can see their story book library. If not, the user will be notified that the credentials are invalid.

Use Case 3 - Uploading Book (User / Caretaker)

*As a user, I want to upload a book so I can create a VSD storybook.*
  1. The user is on the library page.
  2. The user selects “Upload Book.”
  3. The user chooses a file (PDF/image book).
  4. System uploads and stores the book.
  5. System opens the book in VSD Edit Mode.

Use Case 4 - Edit VSD Pages (User / Caretaker)

*As a user, I want to add interactive vocabulary to the pages.*
  1. The user enters Edit Mode for a book.
  2. The user selects objects on a page.
  3. The user assigns a word to each object.
  4. The user can comment on pages.
  5. The user saves the VSD layout.
  6. System will store changes in the library.

Use Case 5 - Save and Manage Storybook Library (User / Caretaker)

*As a user, I want to save and reopen books so I can edit them later.*
  1. The user selects “Save Book.”
  2. System will store the story book in the user’s library.
  3. The user can reopen the books anytime.
  4. The user can rename, edit, or delete saved books.

Use Case 6 - Pick and Read a Book (Reader / AAC User)

*As a user, I want to be able to pick and read a book from my library.*
  1. The user is presented a list of books they’ve previously uploaded.
  2. The user picks a book from the list which opens the book to the cover/first page.
  3. The user reads the page and clicks on any (highlighted) pictures which provide a text-to-speech word of the picture.
  4. The user clicks on the edge of the page or an arrow button to flip the page.

Use Case 7 - Play Text-To-Speech Audio (Reader / ACC User)

*As a reader, I want to hear the words read out loud.*
  1. The user selects a VSD object.
  2. System uses text-to-speech to play the work from the object.
  3. The user can repeat the audio by clicking/tapping again.

Use Case 8 - Export and Share Books (User / Caretaker)

*As a user, I want to share and export my storybooks.*
  1. The user selects a saved book.
  2. The user selects “Export” or “Share.”
  3. System creates a downloadable file.
  4. The user can send the book to another user.

Use Case 9 - Close Application (User)

*As a user, I want to log out or exit the application.*
  1. The user selects “Log Out” or closes the application.
  2. System saves current progress.
  3. The user exits the application.
